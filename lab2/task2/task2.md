# Запрещаем

## 1) Запретите пользователю user1 из предыдущего задания выполнять вход в систему
sudo usermod -s /usr/sbin/nologin user1
# или
sudo usermod -L user1

## 2) Как вы это сделали?
Изменил оболочку пользователя на /usr/sbin/nologin через usermod. При попытке входа система завершает сеанс, так как nologin не является командным интерпретатором.

## 3) Какие ещё способы это сделать вы знаете?
1. `sudo passwd -l user1` - блокировка пароля (добавляет ! к хешу в /etc/shadow)
2. `sudo usermod -e 1970-01-01 user1` - установка прошедшей даты истечения
3. Ручное добавление ! или * перед паролем в /etc/shadow
4. `sudo chage -E 0 user1` - немедленная блокировка через chage

## 4) Можно ли создать пользователей с одинаковыми username?
Нет. Имя пользователя должно быть уникальным в системе. Команда `sudo useradd user1` выдаст ошибку если user1 уже существует.
