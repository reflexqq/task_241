# Ответы на вопросы по файловым системам

## 1. Какие файловые системы вы знаете?
- **Windows**: NTFS, FAT32, exFAT, ReFS
- **Linux/Unix**: ext4, ext3, ext2, XFS, Btrfs, ZFS, JFS
- **macOS**: APFS, HFS+
- **Сетевые**: NFS, CIFS/SMB, SSHFS
- **Специальные**: procfs, sysfs, tmpfs, devtmpfs

## 2. Как можно классифицировать файловые системы? В чём отличия?
- **По назначению**:
  - Дисковые (ext4, NTFS) — для хранения на дисках
  - Сетевые (NFS, SMB) — для доступа по сети
  - Виртуальные (procfs, sysfs) — для доступа к ядру системы
- **По структуре**:
  - Журналируемые (ext4, NTFS) — ведут журнал изменений
  - Нежурналируемые (FAT32) — нет журнала, быстрее, но менее надёжны
- **По поддержке прав**:
  - UNIX-стиль (права rwx)
  - Windows ACL (списки контроля доступа)

## 3. Какие файловые системы используются в Linux?
- **Основные**: ext4 (стандарт), XFS (для больших файлов), Btrfs (снапшоты)
- **Загрузочные**: ext2/ext3 (для /boot)
- **Виртуальные**: procfs, sysfs, tmpfs, devtmpfs
- **Сетевые**: NFS, CIFS
- **Экзотические**: ZFS, F2FS (для SSD)

## 4. Как можно создать файловую систему на диске?
# 1) Разметка диска (например, /dev/sdb1)
sudo fdisk /dev/sdb

# 2) Создание файловой системы
sudo mkfs.ext4 /dev/sdb1  # для ext4
# или
sudo mkfs.xfs /dev/sdb1   # для XFS
## 5. Как можно подключить диск в систему, что такое монтирование?
**Монтирование** — это процесс подключения файловой системы к определённой точке в дереве каталогов.

# Монтирование вручную
sudo mount /dev/sdb1 /mnt/mydisk

# Автомонтирование через /etc/fstab
# Добавить строку:
# /dev/sdb1 /mnt/mydisk ext4 defaults 0 2
## 6. Файловая система procfs, cifs, tmpfs, sysfs. В чём особенности каждой из них?

- **procfs** (`/proc`) – виртуальная файловая система, предоставляющая информацию о процессах, ядре и системных ресурсах в реальном времени.
- **cifs** (Common Internet File System) – сетевая файловая система, используемая для доступа к общим ресурсам Windows (SMB/CIFS).
- **tmpfs** – файловая система, размещаемая в оперативной памяти; используется для временных файлов, обеспечивает высокую скорость.
- **sysfs** (`/sys`) – виртуальная ФС, отображающая информацию об устройствах, драйверах и настройках ядра.

**Вывод каталогов, к которым примонтированы эти файловые системы:**
mount | grep -E 'procfs|cifs|tmpfs|sysfs'
# или
findmnt -t procfs,cifs,tmpfs,sysfs
## 7. Как можно получить информацию о системе используя лишь команду cat? Вывести информацию о процессоре и состоянии памяти системы.

**Информация о процессоре**
cat /proc/cpuinfo
**Состояние памяти:**
cat /proc/meminfo
Отображает данные об использовании памяти: общий объём, свободная память, кэш, буферы, своп и другие параметры.
**Другие примеры получения системной информации через /proc:**
# Загрузка системы (средние значения за 1, 5 и 15 минут)
cat /proc/loadavg

# Время работы системы (в секундах)
cat /proc/uptime

# Версия ядра и компилятора
cat /proc/version

# Список PCI-устройств
cat /proc/bus/pci/devices

# Информация о подключённых USB-устройствах
cat /proc/bus/usb/devices

# Состояние файловых систем
cat /proc/mounts

# Параметры ядра
cat /proc/sys/kernel/ostype
