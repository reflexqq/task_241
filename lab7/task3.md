Ключики

1. Что такое ssh ключи и зачем они нужны?
SSH-ключи — это пара криптографических ключей (публичный и приватный) для безопасного подключения к серверам без ввода пароля. Приватный ключ хранится у пользователя, публичный — на сервере.
2. Как их создать?
Команда: ssh-keygen -t ed25519 -C "student@S-vm-206" -f ~/.ssh/id_ed25519
3. Создай пару публичный/приватный ключ ed_25519, где они хранятся?
Приватный ключ: ~/.ssh/id_ed25519
Публичный ключ: ~/.ssh/id_ed25519.pub
Проверка: ls -la ~/.ssh/id_ed25519*
4. Скопируйте публичный ключ на ваш сервер, в каком файле он будет храниться?
Файл: ~/.ssh/authorized_keys
Команда копирования: ssh-copy-id -i ~/.ssh/id_ed25519.pub student@IP_СЕРВЕРА
5. Попробуйте подключиться к серверу, у вас запросили пароль?
Подключение: ssh -i ~/.ssh/id_ed25519 student@IP_СЕРВЕРА
Если запросили пароль — ключ не настроен правильно
6. Запретите подключение с паролем для всех пользователей, оставьте только с помощью ключа.
Редактируем конфиг: sudo nano /etc/ssh/sshd_config
Меняем: PasswordAuthentication no
